vclaw.

A local-first fleet of AI-controlled machines. Install a node on each computer I own — then chat with its AI, drive its screen, and run commands from one console, anywhere. The story and the build, in one room.

Why I built it

I run several machines and wanted one assistant that could actually operate them — open apps, move files, run commands, reboot a box — from my phone, from anywhere. OpenClaw showed the shape of it, but the usual route (a Telegram bot or a hosted service) traded away two things I wasn’t willing to give up.

Privacy. Every message and file would pass through someone else’s servers — for an agent that reads my files and runs my shell, the wrong default.
Dependency. Their downtime is my downtime. I didn’t want a controller I didn’t control.

So VClaw runs on my hardware — no middleman, no provider to depend on — and is still reachable from outside my network over my own Cloudflare WARP mesh, with me approving anything risky. Read the full note →

AG-UI-style HILAn event protocol so the agent asks with real components (multi-select, forms) and gets structured answers back.
Capability + approvalEvery risky action is gated and waits for a tap. Secrets never enter the model or child processes.
WARP, not a public botRemote access over a private Cloudflare mesh — reachable anywhere, nothing exposed to the internet.
